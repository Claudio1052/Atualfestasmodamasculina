TERMO DE RESPONSABILIDADE E COMPROMISSO — ATUAL FESTAS MODA MASCULINA

Ao realizar a locação de trajes e acessórios junto à ATUAL FESTAS MODA MASCULINA, o locatário declara ter lido, compreendido e concordado integralmente com as seguintes cláusulas:

1. CUIDADO E CONSERVAÇÃO DA PEÇA: O locatário compromete-se a zelar pelo traje com o máximo de cuidado durante todo o período de uso, tratando a peça como bem alheio de grande valor sentimental e financeiro, evitando manchas, rasgos, danos por mau uso ou descuido.

2. RESPONSABILIDADE POR DANOS: Em caso de danos materiais ao traje — incluindo manchas permanentes, rasgos, queimaduras, odores impossíveis de remover, perda de peças ou acessórios — o locatário responsabiliza-se integralmente pelo custo de reparo ou reposição, conforme avaliação técnica da loja.

3. PRAZO DE DEVOLUÇÃO: O locatário compromete-se a devolver o traje impreterivelmente até a data acordada no ato da locação. Atrasos geram multa de R$ 50,00 por dia de atraso, além de eventual indenização por impedimento de nova locação.

4. CONDIÇÕES DE DEVOLUÇÃO: O traje deve ser devolvido em condições equivalentes às do recebimento — limpo, passado (quando entregue assim) e acondicionado adequadamente. A loja realizará avaliação na devolução e qualquer discrepância será documentada.

5. PROIBIÇÕES DE USO: É estritamente proibido o uso do traje para fins que exponham as peças a riscos desnecessários, como ambientes com fumaça excessiva, atividades físicas intensas, chuva sem proteção, ou qualquer situação que possa comprometer a integridade da peça.

6. HIGIENE E LIMPEZA: O locatário não deve lavar, secar na máquina ou passar as peças sem orientação da loja. A limpeza profissional é de responsabilidade da ATUAL FESTAS MODA MASCULINA após a devolução, exceto em casos de sujeira excessiva causada por descuido do locatário.

7. TRANSFERÊNCIA PROIBIDA: O traje não pode ser emprestado, sublocado ou cedido a terceiros durante o período de locação. O contrato é pessoal e intransferível.

8. AUTORIZAÇÃO DE COBRANÇA: Em caso de não devolução ou danos irreparáveis, o locatário autoriza a ATUAL FESTAS MODA MASCULINA a adotar as medidas legais cabíveis para recuperação do bem ou ressarcimento dos danos, incluindo registro de boletim de ocorrência.

9. VERIFICAÇÃO PRÉVIA: O locatário confirma que verificou o traje no ato da retirada e que está de acordo com seu estado de conservação, sob pena de não poder alegar, posteriormente, qualquer dano pré-existente não apontado no momento da retirada.

10. ACEITAÇÃO INTEGRAL: A assinatura deste termo — mesmo em formato digital/eletrônico — tem plena validade jurídica, vinculando o locatário a todas as cláusulas acima descritas, em conformidade com o Código Civil Brasileiro e a Lei nº 12.965/2014 (Marco Civil da Internet).

🛠️ Configuração SQL
Execute estes comandos no Supabase → SQL Editor
⚠️ Por que isso é necessário?
O Supabase usa RLS (Row Level Security) que bloqueia INSERT/UPDATE/DELETE por padrão para a chave anônima. Execute os SQLs abaixo para liberar as permissões necessárias.
PASSO 1 — Permissões da tabela roupas
-- Habilitar RLS na tabela roupas (caso não esteja)
ALTER TABLE roupas ENABLE ROW LEVEL SECURITY;

-- Permitir leitura pública (catálogo visível para todos)
DROP POLICY IF EXISTS "roupas_select_public" ON roupas;
CREATE POLICY "roupas_select_public"
  ON roupas FOR SELECT
  TO anon, authenticated
  USING (true);

-- Permitir INSERT para anon (admin do site)
DROP POLICY IF EXISTS "roupas_insert_anon" ON roupas;
CREATE POLICY "roupas_insert_anon"
  ON roupas FOR INSERT
  TO anon
  WITH CHECK (true);

-- Permitir UPDATE para anon
DROP POLICY IF EXISTS "roupas_update_anon" ON roupas;
CREATE POLICY "roupas_update_anon"
  ON roupas FOR UPDATE
  TO anon
  USING (true)
  WITH CHECK (true);

-- Permitir DELETE para anon
DROP POLICY IF EXISTS "roupas_delete_anon" ON roupas;
CREATE POLICY "roupas_delete_anon"
  ON roupas FOR DELETE
  TO anon
  USING (true);
PASSO 2 — Adicionar coluna imagem_url (se não existir)
-- Adicionar coluna imagem_url na tabela roupas
ALTER TABLE roupas
  ADD COLUMN IF NOT EXISTS imagem_url TEXT;

-- Verificar a estrutura atual da tabela
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'roupas'
ORDER BY ordinal_position;
PASSO 3 — Permissões do Storage (bucket imagens)
-- Permitir upload de imagens para usuários anônimos
INSERT INTO storage.buckets (id, name, public)
  VALUES ('imagens', 'imagens', true)
  ON CONFLICT (id) DO UPDATE SET public = true;

-- Política de leitura pública
DROP POLICY IF EXISTS "imagens_public_read" ON storage.objects;
CREATE POLICY "imagens_public_read"
  ON storage.objects FOR SELECT
  TO public
  USING (bucket_id = 'imagens');

-- Política de upload para anon
DROP POLICY IF EXISTS "imagens_anon_upload" ON storage.objects;
CREATE POLICY "imagens_anon_upload"
  ON storage.objects FOR INSERT
  TO anon
  WITH CHECK (bucket_id = 'imagens');

-- Política de update para anon
DROP POLICY IF EXISTS "imagens_anon_update" ON storage.objects;
CREATE POLICY "imagens_anon_update"
  ON storage.objects FOR UPDATE
  TO anon
  USING (bucket_id = 'imagens');

-- Política de delete para anon
DROP POLICY IF EXISTS "imagens_anon_delete" ON storage.objects;
CREATE POLICY "imagens_anon_delete"
  ON storage.objects FOR DELETE
  TO anon
  USING (bucket_id = 'imagens');
✅ Como aplicar:
1. Acesse supabase.com → seu projeto
2. Menu esquerdo → SQL Editor
3. Cole e execute cada bloco acima na ordem (1, 2, 3)
4. Volte aqui e tente adicionar/editar um traje
